## 三、已修复的 Bug

### Bug 1: 服务端 GUI 状态不更新
- **现象**: 服务启动后，状态显示"未启动"，停止按钮灰色
- **原因**: `FrpServer.start()` 中 `await self._server.serve_forever()` 阻塞了事件循环
- **修复**: 改为 `asyncio.ensure_future(self._server.serve_forever())`

### Bug 2: 打包后出现黑框
- **现象**: GUI 程序启动时出现控制台黑框
- **原因**: PyInstaller 默认打包为控制台应用
- **修复**: 添加 `--windowed` 参数

### Bug 3: 服务安装失败
- **现象**: `sc create` 命令报错
- **原因**: PowerShell 中 `sc` 是 `Set-Content` 的别名
- **修复**: 将所有 `sc` 改为 `sc.exe`

### Bug 4: 服务启动失败 (错误 1053)
- **现象**: `sc.exe start` 返回错误 1053
- **原因**: Python 服务没有向 Windows 服务管理器报告状态
- **修复**: 修改 GUI 通过检查端口监听状态来判断服务是否运行

### Bug 5: GUI 显示服务未运行（实际已运行）
- **现象**: 服务实际在运行，但 GUI 显示"服务已停止"
- **原因**: `_refresh_status()` 只检查 Windows 服务状态
- **修复**: 添加 `_check_port_listening()` 方法，同时检查端口监听状态

## 四、待解决的问题

### 问题: 停止服务超时
- **现象**: 点击"停止服务"后显示"服务停止超时，请手动结束进程"
- **原因**: `sc.exe stop` 对我们的 Python 服务不起作用（服务没有实现 Windows 服务控制接口）
- **解决方案**: 需要修改 `_stop_service()` 方法，直接通过端口找到 PID 并用 `taskkill` 终止进程

### 修改代码位置
文件: `frps_service_manager.py`

需要修改 `_stop_service()` 方法：

```python
def _stop_service(self):
    """停止服务"""
    exe_dir = os.path.dirname(sys.executable) if getattr(sys, 'frozen', False) else os.path.dirname(__file__)
    config_path = os.path.join(exe_dir, "frps_service_config.json")
    
    bind_port = 7000
    try:
        if os.path.exists(config_path):
            with open(config_path, 'r', encoding='utf-8') as f:
                config = json.load(f)
                bind_port = int(config.get("bind_port", 7000))
    except:
        pass
    
    # 获取监听端口的 PID
    success, stdout, _ = self._run_command(f'netstat -ano | Select-String ":{bind_port}" | Select-String "LISTENING"')
    if success and "LISTENING" in stdout:
        # 解析 PID (最后一列)
        import re
        match = re.search(r'LISTENING\s+(\d+)', stdout)
        if match:
            pid = match.group(1)
            # 强制终止进程
            self._run_command(f"taskkill /f /pid {pid}")
            
            import time
            for i in range(10):
                time.sleep(1)
                if not self._check_port_listening(bind_port):
                    messagebox.showinfo("成功", "服务停止成功！")
                    self._refresh_status()
                    return
    
    messagebox.showerror("失败", "服务停止超时，请手动结束进程。")
    self._refresh_status()
```

## 六、使用说明

### GUI 版本
1. 运行 `frps_lite_gui.exe` 启动服务端
2. 配置绑定地址、端口和 Token
3. 点击"启动服务"
4. 运行 `frpc_lite_gui.exe` 启动客户端
5. 配置服务器地址、端口、Token 和代理规则
6. 点击"连接"

### 系统服务版本
1. 以管理员身份运行 `frps_service_manager.exe`
2. 点击"安装服务"
3. 配置端口和 Token
4. 点击"启动服务"
5. 服务会在后台运行，开机自动启动

## 七、技术要点

### 1. asyncio 事件循环管理
- 使用 `AsyncEngine` 类在后台线程管理事件循环
- `start()` 方法使用 `asyncio.ensure_future()` 避免阻塞

### 2. Windows 服务
- 使用 `sc.exe create/delete/start/stop` 管理服务
- 通过端口监听状态判断服务是否真正运行
- PowerShell 中必须使用 `sc.exe` 而非 `sc`

### 3. PyInstaller 打包
- `--onefile` 单文件打包
- `--windowed` 无控制台窗口
- `--clean` 清理缓存
